Overview

Nile.com (NileDotCom) is a desktop GUI application built in Java that simulates a simple online ordering workflow. Users can search for items, enter quantities, review item details and subtotals, add multiple items to a cart, and finalize an order.

The project emphasizes building a working GUI using Eclipse WindowBuilder while connecting UI actions to backend logic — including calculations, state tracking across workflow steps, and structured file output.

Language Java
UI Framework Java Swing
IDE Eclipse IDE
GUI Designer WindowBuilder
Output transactions.csv
Pattern Event-Driven (ActionListener)

Key Features

Item Search Workflow Lookup tied to button click events — fully event-driven UI with ActionListener bindings.
Quantity & Subtotal Logic Quantity input fields, real-time item detail display, and running subtotal calculations.
Multi-Item Cart Cart display within the GUI supporting multiple items before order completion.
Order Confirmation Step-by-step order confirmation and completion flow with clear UI state transitions.
Transaction Logging Completed orders written to a transactions.csv file — practical file I/O.
Input Validation Graceful handling of invalid values — empty fields, bad quantities, and edge cases.
03

Implementation Notes

🎨
Visual Design with WindowBuilder
The interface was laid out visually using Eclipse's WindowBuilder plugin, allowing drag-and-drop placement of Swing components (JFrame, JPanel, JTextField, JButton, JLabel) before wiring them to logic.
⚡
Event-Driven Wiring
Each button was connected to a Java ActionListener, triggering the appropriate logic — search, add to cart, confirm, and finalize. The separation between UI layout and event handling kept the code organized.
🔄
State Management
Cart state — item count, running total, current item — had to be carefully tracked across UI interactions. GUI fields update consistently as the user moves through each order step.
📄
CSV Transaction Output
On order completion, transaction data is formatted and written to a CSV file using Java file I/O. This reinforced structured output formatting and working with the filesystem from within a GUI application.
