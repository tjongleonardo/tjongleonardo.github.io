Nile.com (NileDotCom)

Java Swing Ordering GUI built with Eclipse WindowBuilder

Overview

Nile.com (NileDotCom) is a desktop GUI application built in Java that simulates a simple online ordering workflow. Users can search for items, enter quantities, review item details and subtotals, add multiple items to a cart, and finalize an order. The project emphasizes building a working GUI using Eclipse WindowBuilder while connecting UI actions to backend logic like calculations and file output.

Images

Key Features

  • Item lookup/search workflow tied to a button click (event-driven UI)
  • Quantity input, item detail display, and subtotal calculation
  • Multi-item cart display within the GUI
  • Order confirmation and completion steps
  • Transaction logging to a transactions.csv output file

Implementation Notes

The interface was designed visually using WindowBuilder, then wired to Java event handlers (e.g., ActionListener) so each button triggers the appropriate logic. The project required careful state management for cart progress (current item count, running total, etc.) and consistent updates to GUI fields as the user moved through the order steps.

The completed order is recorded by writing transaction lines to a CSV file, reinforcing practical file I/O and output formatting.

Tools & Methods

Java, Swing (JFrame, JPanel, JTextField, JButton, JLabel), Eclipse IDE, WindowBuilder, event-driven programming (ActionListener), input validation, file I/O (CSV writing)

Skills Gained

  • GUI development: Building desktop interfaces with Swing and WindowBuilder
  • Event-driven programming: Connecting buttons and inputs to application logic
  • State management: Tracking cart items, totals, and workflow steps across the UI
  • Input validation: Handling invalid values gracefully (quantity, missing fields, etc.)
  • File output: Writing structured transaction records to a CSV file
  • Debugging UI issues: Fixing layout/visibility problems between WindowBuilder and runtime
